Summary

Create Facebook App

We need to have a Facebook App ID to make API calls to Facebook.

Steps

  1. Sign in to your Facebook account
  2. Visit http://developers.facebook.com and click "Add a New App" under "My Apps"

  3. First you need to first add the product, Facebook Login, under the "PRODUCTS".


    After that you can modify the Valid OAuth redirect URIs in order to build the trust between your website and Facebook. Thus, Facebook can redirect the user back to your website.

Facebook Login

This is an example of using Facebook API. API, Application Programming Interface, provides different modules for developers to build up a whole program without requiring to know how it works inside. In other words, API can simplify the process of programming by abstracting the implementation and leaving the objects that is needed for the developers.

The following code is to ask for users' permission to log in onto their Facebook account and authorize this page to get and use some of their information.

In the HTML, you can first create a button to prompt for the permissions dialog with the fb:login-button tag. You need to add the permissions list in the scope attribute. The onlogin="checkLoginState()" will call the checkLoginState function when the users log in successfully.

Steps

  1. Check the login status
  2. Invoke the login dialogue and ask for permission
  3. Successful login
  4. Access and apply the users' personal information with authorization

After initializing the Facebook SDK, the page will check the login status by calling the FB.getLoginStatus(), then it passes the status response to the statusChangeCallback(response) to check the current login status. If the status is "connected", which means the users has logged into their account at the moment, the testAPI() will be called. If the status from the response is stated as "not_authorized", it implies that the users have not authorized your application to log in with their Facebook account. Other situations will show the notice for asking the users to log into their Facebook.

When the users try to log into their Facebook account, a login dialogue will pop up and ask for the users' permission to access their personal information. When the users authorize you to do so, Facebook Server will send the access token to your application. Your application will store the access token for sending back to Facebook server whenever it needs to make the API call. That said, you will not aware of this access token processing as this will be done within the SDK. This authorization mechanism is called OAuth, widely used for providing personal information with the advantage of disclosing their usernames and passwords

Successful Response

In order to understand what responses that are actually sent back, you can insert the console.log(response) under the statusChangeCallback() and testAPI(). The image above shows the two responses. The upper response object with the current status "connected". The bottom response object with the user's id and the profile name of the user.

To access the response, you can simply type [object.name].[attribute] that is response.name for this given example.
